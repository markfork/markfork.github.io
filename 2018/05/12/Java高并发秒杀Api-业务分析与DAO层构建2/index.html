DAO层接口设计与SQL编写的反思

DAO - data access object 数据访问层的简称
这一层我们主要做的是核心数据操作的接口声明，以及SQL编写，并没有涉及到Service 业务逻辑层代码的编写。这样做的好处是什么呢？
主要有以下三点

  • 代码分层，DAO层只关注于核心数据操作的接口声明&SQL编写
  • 代码和SQL的分离，方便代码review
  • 业务逻辑层主要做事务控制、事务中完成DAO层的代码组合与拼接。每一层都可以做单元测试。
